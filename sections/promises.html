Promises

A Promise in JavaScript represents an eventual completion (or failure) of an asynchronous operation and its resulting value. It allows you to handle asynchronous tasks in a more manageable way than callbacks.

States of a Promise:

  • Pending: Initial state, neither fulfilled nor rejected.
  • Fulfilled: The operation completed successfully.
  • Rejected: The operation failed.

You can attach handlers to a Promise to perform actions when the promise is fulfilled or rejected, using .then() and .catch().

Example for Kids

You ask your mom if you can have a cookie after dinner. She tells you:

"If you eat all your dinner, I promise you'll get a cookie!" So now you have to wait and see what happens:

If you eat all your dinner, your mom says, "Yay! You get a cookie!" That's like the promise being kept.

If you don't eat your dinner, your mom says, "No cookie for you!" That's like the promise being broken.

A promise in JavaScript is just like that! It says, "I'll do something later," and you wait to see if it works or not.


// Creating a Promise
const myPromise = new Promise((resolve, reject) => {
  const success = true;

  if (success) {
    resolve("Promise was successful!");
  } else {
    reject("Promise failed.");
  }
});

// Using the Promise
myPromise
  .then(result => {
    console.log(result); // Output: Promise was successful!
  })
  .catch(error => {
    console.log(error); // If rejected, output: Promise failed.
  });

Chaining Promises:

Promises can be chained to handle multiple asynchronous operations sequentially.


fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => {
    console.log(data);
    return data;
  })
  .catch(error => {
    console.error('Error:', error);
  });
