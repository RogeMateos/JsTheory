Setting Default Values

Logical OR (`||`) for Default Values

JavaScript allows you to provide default values for function parameters, even if they aren't provided. By using the logical OR (`||`) operator, we can assign a fallback value when the argument passed to the function is `undefined`, `null`, or another falsy value.

This is a useful technique to ensure that your function behaves predictably even when the caller does not provide all of the arguments.

Basic Example:


        function greet(name) {
          name = name || 'Your name here';
          console.log('Hello ' + name);
        }
  
        greet('Tony'); // Output: Hello Tony
        greet();       // Output: Hello Your name here

In this example, the `greet` function will output "Hello Tony" if the `name` parameter is provided. If not, it will use the default value "Your name here". This works because the OR operator returns the first "truthy" value it encounters.

How the OR Operator Works:

The OR (`||`) operator in JavaScript is more than just a boolean operator. It evaluates the operands from left to right and returns the first operand that is "truthy". If none of the values are "truthy," it returns the last one.


  // Example 1
  console.log(true || false);  // Output: true
  
  // Example 2
  console.log(undefined || 'hello'); // Output: hello
  
  // Example 3
  console.log(0 || 'default');  // Output: default

In the first example, `true || false` returns `true`, because `true` is a "truthy" value. In the second example, `undefined || 'hello'` returns `'hello'`, because `undefined` is falsy, and `'hello'` is truthy. In the third example, `0 || 'default'` returns `'default'` because `0` is falsy.

Default Value Assignment in Functions:

When a function parameter is `undefined`, we can use the OR operator to provide a default value. This technique is commonly used in JavaScript libraries and frameworks. In this scenario, if the `name` parameter is missing or falsy, it will fall back to the default string 'Your name here'.


        function greet(name) {
          name = name || 'Your name here';
          console.log('Hello ' + name);
        }
  
        greet('Tony');       // Output: Hello Tony
        greet(undefined);    // Output: Hello Your name here

The OR operator evaluates `name`, and if it is `undefined`, `null`, or an empty string, the default value ('Your name here') will be used instead. This way, the function behaves gracefully even when arguments are missing.

Potential Pitfalls:

While this pattern is useful, there are some cases where it might not work as expected. For example, `0`, `false`, or an empty string `''` are all falsy values, so they will trigger the default value:


        greet(0);  // Output: Hello Your name here

This happens because `0` is considered falsy, so it falls back to the default value. If you need to allow such values, you'll need to handle those cases more explicitly or use the nullish coalescing operator (`??`) introduced in ES2020, which only falls back when `undefined` or `null` is encountered:


        function greet(name) {
          name = name ?? 'Your name here';
          console.log('Hello ' + name);
        }
  
        greet(0);  // Output: Hello 0

The nullish coalescing operator (`??`) only triggers the default value when the value is `null` or `undefined`, making it a more suitable alternative in some cases.
