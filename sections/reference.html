Reference and Value

Value doesn't change when we reassign and reference if we change one object it changes the other.

Value

In JavaScript, if we assign a variable to a primitive type, the value is copied. But if it is not a primitive, we copy the reference to the value.

And what happens if we copy the reference instead of the value? Both objects point to the same location, so if we modify one, the other will also be affected.

These include types like string, number, boolean, null, undefined, symbol, and bigint.

When you assign a primitive value to a variable, JavaScript copies the value itself. This means that if you create a new variable and assign it the value of an existing primitive variable, they are completely independent of each other.


let a = 10;
let b = a; // b is a copy of the value in a
b = 20;
console.log(a); // Output: 10
console.log(b); // Output: 20

Reference

However, when you assign a reference type (such as an object or an array) to a variable, JavaScript copies the reference to the value. If you assign this reference to another variable, both variables point to the same memory location. Changing the object via either variable will affect the other.

Let's understand this with an example:


let obj1 = { name: 'Alice' };
let obj2 = obj1; // obj2 now holds a reference to the same object
obj2.name = 'Bob';
console.log(obj1.name); // Output: Bob
console.log(obj2.name); // Output: Bob

Conclusion

  • Primitive types are copied by value.
  • Reference types are copied by reference.

Scope

Scope in JavaScript

Scope in JavaScript refers to the current context of code, which determines the accessibility of variables to JavaScript. There are three main types of scope:

1. Global Scope

  • Variables declared outside of any function or block have global scope.
  • They can be accessed from anywhere in the JavaScript program.
  • In a browser, the global scope is usually the window object.

var globalVar = "I'm global";

function someFunction() {
    console.log(globalVar); // Accessible
}

console.log(globalVar); // Also accessible

2. Local Scope (Function Scope)

  • Variables declared inside a function have local scope.
  • They can only be accessed within that function.
  • Each function creates its own scope.

function someFunction() {
    var localVar = "I'm local";
    console.log(localVar); // Accessible
}

console.log(localVar); // Error: localVar is not defined

3. Block Scope (introduced with ES6)

  • Variables declared inside a block (denoted by curly braces {}) using let or const have block scope.
  • They can only be accessed within that block.
  • This includes blocks in if statements, for loops, etc.

if (true) {
    let blockVar = "I'm block-scoped";
    console.log(blockVar); // Accessible
}

console.log(blockVar); // Error: blockVar is not defined

Key Points

  • Variables with global scope can be accessed from anywhere, but this can lead to naming conflicts and is generally considered bad practice if overused.
  • Local scope provides encapsulation and helps avoid naming conflicts.
  • Block scope allows for more precise control over variable visibility and lifetime.
  • The let and const keywords respect block scope, while var does not (it only respects function scope).
