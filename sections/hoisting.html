Hoisting

Hoisted definition

Hoisting means that variable and function declarations are moved to the top of their execution context before the code is run.

When the function executes, the compiler looks for variable declarations, and if the variable is declared, it hoists it to the top.

JavaScript engine makes multiple passes through the code. In the first pass, it stores function declarations and variables in memory before executing any code.

For variables, only the declarations are hoisted, not the initializations. This means that if you declare a variable after using it, JavaScript wonâ€™t throw an error; instead, the variable will be undefined until it is initialized.


        console.log(x); // undefined
        var x = 5;
        console.log(x); // 5

For functions, both the function declaration and the function expression are hoisted. This means you can call a function before it is declared in the code.


        hello(); // "Hello, world!"

        function hello() {
          console.log("Hello, world!");
        }

In this case, the code works because the entire function declaration `hello` is hoisted to the beginning of the execution context.

Two examples


        console.log(greeting);
        hello();
        function hello(){
            console.log('hello');
        }
        var greeting = "Hi there";
        console.log(greeting);

        // Output:
        // undefined
        // hello
        // Hi there
      

        function hello(){
          console.log('hello');
        }
        var greeting;
        console.log(greeting);
        hello();
        greeting = "Hi there";
        console.log(greeting);

        // Output:
        // undefined
        // hello
        // Hi there
