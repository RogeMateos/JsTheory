JSON vs Object Literals

JSON (JavaScript Object Notation) is inspired by JavaScript object literals but is more strict and designed as a lightweight format for sending data over the internet. Although JSON looks like object literal syntax, they are not exactly the same, and understanding their differences helps prevent common mistakes.

The key difference is that JSON requires property names to be wrapped in quotes and does not support certain JavaScript-specific features like functions or undefined values. 


  // JavaScript Object Literal
  const person = {
      firstName: 'Mary',
      isProgrammer: true
  };
  
  // Convert to JSON string
  const jsonString = JSON.stringify(person);
  console.log(jsonString); 
  // Output: {"firstName":"Mary","isProgrammer":true}
  
  // Convert JSON string back to object
  const jsonValue = JSON.parse(jsonString);
  console.log(jsonValue);
  // Output: { firstName: 'Mary', isProgrammer: true }

In contrast to XML, JSON is more compact and has become the standard format for exchanging data between servers and browsers, as well as in server environments like Node.js.
