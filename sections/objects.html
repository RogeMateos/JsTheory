Objects and Primitive Values

Objects in JavaScript

An object in JavaScript is a complex data type that allows you to store collections of data and more complex entities. Objects can contain properties and methods.

  • They are mutable (can be changed after creation)
  • They are stored and copied by reference
  • They can contain multiple values as properties
  • They can have methods (functions as properties)

      let person = {
        name: "Alice",
        age: 30,
        greet: function() {
          console.log("Hello, I'm " + this.name);
        }
      };
      
      person.greet(); // Output: Hello, I'm Alice

Primitive Values

Primitive values are the simplest data types in JavaScript. They include:

  • String
  • Number
  • Boolean
  • Undefined
  • Null
  • Symbol
  • BigInt

Key characteristics of primitives:

  • They are immutable (cannot be changed after creation)
  • They are stored and copied by value
  • They represent a single value

      let str = "Hello";
      let num = 42;
      let bool = true;
      let undef = undefined;
      let n = null;
      let sym = Symbol("unique");
      let bigInt = 1234567890123456789012345678901234567890n;

Main Differences

  1. Complexity: Primitives are simple, single values; Objects can contain multiple values and have complex structures.
  2. Mutability: Primitives are immutable; Objects are mutable.
  3. Storage and copying: Primitives are copied by value; Objects are copied by reference.
  4. Comparison: Primitives are compared by value; Objects are compared by reference.
  5. Methods: Primitives don't have methods (although JavaScript temporarily wraps them in objects to allow method-like syntax); Objects can have methods.

      // Primitives
      let a = 5;
      let b = a;
      b = 10;
      console.log(a); // 5
      console.log(b); // 10

      // Objects
      let obj1 = {x: 5};
      let obj2 = obj1;
      obj2.x = 10;
      console.log(obj1.x); // 10
      console.log(obj2.x); // 10
